# Как правильно сообщить о проблеме или предложить улучшение  

Спасибо, что решили помочь улучшить LEGO Train Panel!  
Чтобы ваш запрос был рассмотрен максимально быстро и эффективно — пожалуйста, следуйте этим рекомендациям.

## 1. Прежде чем создавать issue

Проверьте, не решена ли уже ваша проблема:

- [x] посмотрите [закрытые issues](https://github.com/EvelRus/LegoTrainPanel/issues?q=is%3Aissue+is%3Aclosed)
- [x] прочитайте [последние коммиты](https://github.com/EvelRus/LegoTrainPanel/commits/main)
- [x] убедитесь, что вы используете **самую свежую версию** проекта (`git pull` или скачали заново архив)

## 2. Какой тип issue вы хотите создать?

| Тип                               | Шаблон выбрать                                                                 | Что писать в заголовке (пример)                                 |
|:--                                |:-------------------------------------------------------------------------------|:----------------------------------------------------------------|
| Баг / не работает                 | 🐛 Bug report                                                                   | [Hub не подключается] Pybricks хаб не виден после перезагрузки |
| Ошибка в интерфейсе / визуальный баг | 🖥 UI / визуальный баг                                                        | [UI] Скорость отображается некорректно после E-STOP             |
| Предложение новой функции         | ✨ Feature / улучшение                                                          | [Feature] Добавить поддержку датчика наклона                   |
| Вопрос / не понятно как работает  | ❓ Question                                                                     | [Question] Как записать сценарий с двумя поездами одновременно? |
| Другое (документация, typo и т.д.)| 📝 Другое                                                                       | [Docs] Опечатка в описании traffic light                        |

## 3. Шаблон для сообщения о баге (рекомендуемый)

Пожалуйста, скопируйте и заполните:

```markdown
## Краткое описание проблемы
(что именно не работает, в одном-двух предложениях)

## Ожидаемое поведение
(что должно было произойти)

## Текущее поведение
(что происходит на самом деле)

## Шаги для воспроизведения
1. Запускаем `node server.js`
2. Подключаем хаб LEGO Technic Hub
3. Нажимаем кнопку ...
4. Видим ...

## Окружение
* ОС: Windows 11 / macOS 15 / Ubuntu 24.04 / …
* Версия Node.js: `node -v` → v20.12.2
* Браузер: Chrome 128 / Safari 18 / …
* Версия проекта: (commit hash или дата последнего `git pull`)
* Тип хаба: PoweredUP Hub / Technic Large Hub / Pybricks / …

## Логи
(очень важно!)

**Из консоли сервера** (та, где запущен `node server.js`):
[2026-03-09T14:35:12.123Z] [WARN] [PyBricks] EVT-характеристика не найдена
[2026-03-09T14:35:14.567Z] [ERROR] write error: characteristic not found

**Из браузерной консоли** (F12 → Console):
WebSocket connection failed: ...
Uncaught TypeError: ...


## Дополнительно (по желанию, но очень помогает)

- скриншот / видео (можно загрузить прямо в issue)
- фото реальной сборки / расположения датчиков (если баг связан с сенсорами)
- содержимое файла `hub-config.json` (удалите личные данные, если есть)
- какие именно наборы LEGO / моторы / датчики используются

## 4. Хороший заголовок issue — это уже 50% успеха

Плохие варианты:  
- не работает  
- баг  
- ошибка  
- помогите пожалуйста

Хорошие варианты:  
- [Pybricks] Ошибка 133 при записи в stdin после долгого простоя  
- [UI] Ползунок скорости не реагирует после переподключения хаба  
- [Feature] Добавить кнопку «Сбросить все настройки хаба»  
- [Bug] Сценарий зависает на шаге colorSensor если timeout=0
```
Качественно оформленный issue сильно ускоряет исправление.
