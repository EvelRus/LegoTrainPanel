# 🚂 LEGO Train Panel
Веб-приложение для управления поездами LEGO PoweredUP через браузер.  
Управляйте несколькими поездами одновременно с любого устройства в локальной сети — ПК, iPad, смартфон.

> Специально для **«Лего Малефик»** — © Шарапов В.Н., 2026

---

## ✨ Возможности

### 🎮 Управление поездами
- **Автообнаружение** — поезда появляются в интерфейсе автоматически при включении хаба
- **Плавный рамп** — настраиваемый шаг скорости и интервал исключают механические рывки и «зависание» хаба TrainMotorLarge
- **Ползунок скорости** с центральной точкой (±8% → автоснап в 0)
- **Кнопки**: −100, −10, ■ СТОП, +10, +100, ▶ СТАРТ 20%, ■ ФИНИШ
- **3 пресета скорости** — настраиваемые кнопки быстрого доступа (по умолчанию 20/50/80%)
- **Гудок** — удержание кнопки воспроизводит звук, отпускание останавливает
- **Спидометр** — SVG-стрелка с цветовым градиентом (синий ← → красный)
- **Фото поезда** с виньеткой на карточке

### 🛑 Безопасность
- **E-STOP** — одна кнопка в шапке останавливает ВСЕ поезда немедленно, отменяя рампы и сценарии. Пульсирует пока хоть один поезд движется
- **Автостоп** — при закрытии всех браузеров сервер останавливает поезда
- **Watchdog** — рамп, идущий более 15 с, автоматически прерывается

### 🔗 Составы
- Объединить несколько хабов (например, два локомотива 60198 + 60197)
- Один ползунок управляет всем составом синхронно
- Поезда в составе отмечаются фиолетовым бейджем

### 🎬 Сценарии
- **Запись в реальном времени** — нажмите «Запись», управляйте вручную, нажмите «Стоп»
- Все команды с реальными паузами сохраняются и точно воспроизводятся
- Управление: старт ▶ / стоп ■ / удалить 🗑

### 🕐 Расписание
- Запуск действий по времени (ЧЧ:ММ) и дням недели
- Действия: E-STOP / установить скорость поезда / запустить сценарий / управление составом
- Проверка каждую минуту

### 📡 Сенсоры (опционально)
- Автоопределение цветового сенсора / датчика расстояния на порту B
- Данные отображаются в реальном времени на карточке поезда

### 💡 LED хаба
- Кнопка питания хаба меняет цвет автоматически: 🟢 вперёд / 🔵 назад / 🔴 стоп

### 🔊 Звуки
- 4 звука на поезд: старт, тормоз, гудок, зацикленный звук движения
- Громкость «движения» масштабируется со скоростью
- Кнопка гудка на хабе (кнопка питания) автоматически воспроизводит гудок
- Глобальный мут в шапке

### ⚙️ Настройки поезда
- Название, фото, звуки — через встроенный файловый браузер
- Параметры рампа: шаг (1–25%) и интервал (50–400 мс) с подсказкой «диапазон 200% за ~X с»
- Пресеты скорости

### 📋 Журнал событий
- Выезжающая панель снизу с событиями в реальном времени
- Уровни: INFO / WARN / ERROR / EVENT с цветовой кодировкой
- Файлы логов: `data/logs/YYYY-MM-DD.log` (один файл на сутки)
- API: `GET /api/logs`, `GET /api/logs/files`, `GET /api/logs/file?name=`

### 📱 PWA (Progressive Web App)
- Установка на iPad/iPhone/Android как приложение («Добавить на экран»)
- Полноэкранный режим без адресной строки
- Кеширование статики для работы в локальной сети без интернета

## 🔧 Установка

### Требования

- **Node.js** версии 20 или новее ([nodejs.org](https://nodejs.org))
- **Bluetooth 5.0+** (BLE) — встроенный или USB-адаптер
- **npm** (устанавливается вместе с Node.js)

---

### macOS

```bash
# 1. Установить Node.js (если не установлен)
brew install node
# или скачать с https://nodejs.org

# 2. Клонировать репозиторий
git clone https://github.com/EvelRus/LegoTrainPanel.git
cd LegoTrainPanel

# 3. Установить зависимости
npm install

# 4. Запустить
node server.js
```

Открыть браузер: **http://localhost:3000**

> **macOS Monterey+:** если Bluetooth не определяется, разрешите Node.js доступ к Bluetooth в  
> Системные настройки → Конфиденциальность → Bluetooth

---

### Linux (Ubuntu/Debian/Raspberry Pi)

```bash
# 1. Установить Node.js 20+
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 2. Установить зависимости Bluetooth (если не установлены)
sudo apt-get install -y bluetooth bluez libbluetooth-dev libudev-dev

# 3. Разрешить Node.js работать с BLE без sudo
sudo setcap cap_net_raw+eip $(eval readlink -f `which node`)

# 4. Клонировать репозиторий
git clone https://github.com/EvelRus/LegoTrainPanel.git
cd LegoTrainPanel

# 5. Установить зависимости
npm install

# 6. Запустить
node server.js
```

Открыть браузер: **http://localhost:3000**

#### Для Raspberry Pi — автозапуск при старте

```bash
# Создать systemd-сервис
sudo nano /etc/systemd/system/legotrainpanel.service
```

```ini
[Unit]
Description=LEGO Train Panel
After=bluetooth.target network.target

[Service]
ExecStart=/usr/bin/node /home/pi/LegoTrainPanel/server.js
WorkingDirectory=/home/pi/LegoTrainPanel
Restart=on-failure
User=pi
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable legotrainpanel
sudo systemctl start legotrainpanel
# Проверить статус:
sudo systemctl status legotrainpanel
```

## ⚡ Быстрый старт

### 1. Настройте поезд

При первом подключении хаб автоматически добавляется в `hub-config.json` с именем `Train-1`.

Нажмите **🚃** на карточке поезда → откроется окно настроек:
- Задайте название (например, «Крокодил» или «60197 / Пассажирский»)
- Добавьте фото: положите файл в `public/photo/`, укажите путь `photo/train.jpg`
- Добавьте звуки: положите MP3 в `public/sound/train-name/`, укажите пути

### 2. Структура звуков

```
public/sound/
└── crocodile/          # название поезда (любое)
    ├── start.mp3       # звук при старте (1–3 сек)
    ├── brake.mp3       # звук при торможении
    ├── horn.mp3        # гудок
    └── moving-loop.mp3 # зацикленный звук движения
```

> Рекомендуемый формат: **MP3**, 44100 Гц, стерео или моно.  
> Звуки поездов LEGO можно найти на [freesound.org](https://freesound.org) (поиск: «train steam», «train horn»).

### 3. Параметры рампа

В настройках поезда подберите параметры под конкретную модель:

| Набор | Рекомендуемый шаг | Интервал |
|-------|-------------------|----------|
| 60337 (два мотора) | 5% | 150 мс |
| 60198, 60197 | 10% | 100 мс |
| 71044 Disney | 5% | 200 мс |
| 10227 Creator | 10% | 100 мс |

## 🔌 API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/config` | Конфигурация всех хабов |
| POST | `/api/config` | Сохранить конфигурацию |
| GET | `/api/info` | Версии библиотек, uptime |
| GET | `/api/browse?path=` | Браузер файлов (для выбора звуков/фото) |
| GET | `/api/logs?n=200` | Последние N записей журнала |
| GET | `/api/logs/files` | Список файлов логов |
| GET | `/api/logs/file?name=` | Содержимое конкретного лог-файла |
| GET | `/api/scenarios` | Сохранённые сценарии |
| GET | `/api/schedules` | Расписание |
| GET | `/api/consists` | Составы |

### Socket.IO события (клиент → сервер)

| Событие | Данные | Описание |
|---------|--------|----------|
| `setSpeed` | `{trainId, speed}` | Установить скорость (−100..+100) |
| `estop` | — | Экстренная остановка всех поездов |
| `reconnectHub` | `{trainId}` | Переподключить хаб |
| `setRampParams` | `{trainId, stepSize, stepMs}` | Параметры рампа |
| `savePresets` | `{trainId, presets}` | Пресеты скорости |
| `setLED` | `{trainId, color}` | Цвет LED хаба |
| `saveConsist` | `{id, consist}` | Сохранить состав |
| `setConsistSpeed` | `{consistId, speed}` | Скорость состава |
| `startRecording` | `{name}` | Начать запись сценария |
| `stopRecording` | — | Остановить запись |
| `playScenario` | `{name}` | Воспроизвести сценарий |
| `addSchedule` | `{id, schedule}` | Добавить задачу расписания |

## 🐛 Решение проблем

**Поезд не появляется в интерфейсе**
- Убедитесь что Bluetooth включён на компьютере
- Убедитесь что хаб в режиме BLE (кнопка питания мигает зелёным)
- Linux: проверьте `sudo setcap` (см. установку)
- Windows: перезапустите Bluetooth-службу

**Поезд не реагирует на команды**
- Уменьшите шаг рампа (5%) и увеличьте интервал (150–200 мс) в настройках
- Проверьте уровень заряда батарей (отображается на карточке)
- Убедитесь что поезд в зоне BLE-приёма (~10 м без препятствий)

**Ошибка `Cannot find module './lib/...'`**
- Убедитесь, что папка `lib/` существует рядом с `server.js`
- Структура должна быть: `./LegoTrainPanel/lib/ramp.js`, `./LegoTrainPanel/lib/logger.js`, `./LegoTrainPanel/lib/scheduler.js`

**Браузер не подключается**
- Сервер запущен? Проверьте консоль на наличие строки `🚂 http://localhost:3000`
- Брандмауэр? Разрешите входящие соединения на порт 3000
